# Conventions & Debugging

---

## Code Conventions

### File placement

- All application components go in `components/dataset-manager/` — flat, no sub-folders
- `components/ui/` contains shadcn/ui primitives — never edit these; they are regenerated by the shadcn CLI
- New `lib/` modules go in `frontend/lib/` — no React, pure TypeScript

### Naming

| Thing | Convention | Example |
|-------|-----------|---------|
| Components | PascalCase | `ItemEditWorkspace` |
| Types / interfaces | PascalCase | `TransformConfig` |
| Functions | camelCase | `computeTransformedMetadata` |
| Constants / config objects | UPPER_SNAKE_CASE | `MODEL_CONFIGS` |
| CSS | Tailwind utility classes | `className="flex items-center gap-2"` |

No custom CSS classes except in `globals.css`. Never write inline `style=` unless Tailwind cannot express it (e.g. dynamic percentages).

### Class composition

Always use `cn()` for conditional classes:

```typescript
import { cn } from '@/lib/utils'

className={cn(
  'base-class',
  condition && 'conditional-class',
  isActive ? 'active-class' : 'inactive-class',
)}
```

### Segmented controls

Both `TransformPanel` and `ItemEditWorkspace` have local `SegmentedControl` / `SegCtrl` components with identical implementations. If you change the visual style, update both. Do not extract to a shared component unless you need a third instance.

### Frame rule strings in UI

Never hardcode `=== '8n+1'` in UI components. Use `!== 'any'` and interpolate the rule string:

```tsx
// CORRECT
modelConfig.frames.rule !== 'any'
  ? `Auto ${modelConfig.frames.rule}`
  : 'Auto'

// WRONG
modelConfig.frames.rule === '8n+1' ? 'Auto 8n+1' : 'Auto'
```

This ensures WAN (4n+1) and any future models display correctly without code changes.

## State Patterns

### Never read `datasets` inside an async callback

By the time an async callback fires, `datasets` may have changed (React batched a previous commit). Always collect results synchronously first, then commit once. See [algorithms.md — Stale Closure Pattern](algorithms.md#stale-closure-pattern).

### Modals do not control their own visibility

Modal components receive `open: boolean` and `onClose: () => void` from `DatasetView`. They never call `useState` to track their own open state. `DatasetView` is the single source of truth for which modal is visible.

### Per-item state resets on navigation

`ItemEditWorkspace` has `useEffect([file.id])` that resets `itemConfig`, `splits`, `processState`, `deletedFrames`, and aborts any running job when the user navigates to a different file. If you add new per-item state, add it to this reset block.

### `commit()` is the only way to mutate datasets

If you add a handler in `DatasetManager`, it must call `commit(newDatasets)`. Never call `setDatasets()` directly — that bypasses the history stack and the debounced IndexedDB save.

## Backend Patterns

### Processors are stateless

`process()` receives everything it needs in `ProcessorInput`. No instance state. Multiple jobs can run concurrently without interference.

### Jobs are in-memory

`services/jobs.py` is a simple Python dict. For production (multi-process deployment or persistence across restarts), swap it for Redis. The interface (`create`, `get`, `update`, `delete`) does not change.

### Blocking operations must leave the event loop

FastAPI runs in an async event loop. All `subprocess.run` (ffmpeg) calls inside a processor must be dispatched via `run_in_executor(None, blocking_fn)` to run in a thread pool. The `_background` function in `api/v1/transform.py` already does this.

## Common Bugs

### After Transform shows original values, not transformed

`applyResolution` or `applyFrames` is `false` (toggle is OFF). Check the ON/OFF button state. If the toggle is ON but values still don't change, verify `computeTransformedMetadata` is receiving `itemConfig` (local per-item state), not the global `transformConfig` prop.

### Validation stays "pending" forever

`validateDatasetFile` runs one file at a time via `useEffect`. If a file has `status: 'error'`, it is skipped — only `'pending'` files are retried. Open the browser console and look for errors from `validation.ts`. Common causes: unsupported codec, zero-byte file, or CORS error when loading a blob URL.

### Backend returns 422 Unprocessable Entity

Pydantic validation failure. Run the backend with `--log-level debug` to see the detailed error. Most common cause: the `config` field was sent as a JavaScript object instead of a JSON string. Always `JSON.stringify(config)` before appending to `FormData`.

### Segments are missing or overwritten after batch transform

Stale closure bug. The batch handler is calling `handleReplaceWithSegments` in a loop instead of `handleReplaceBatch` once at the end. Each call in the loop reads stale `datasets` and overwrites the previous result. See [algorithms.md — Stale Closure Pattern](algorithms.md#stale-closure-pattern).

### Blob URL is invalid after page reload

Expected — blob URLs are session-scoped. `loadDatasets()` in `persistence.ts` recreates them from IndexedDB on every page load. If a URL is stale *within* the same session, `URL.revokeObjectURL()` was called too early (e.g. before the `<img>` or `<video>` finished loading).

### FrameGrid shows wrong thumbnails

The frontend `bresenhamSourceMap()` and the backend `resample_bresenham()` must produce identical output. If you change either, verify they still match by running both with the same inputs and comparing. See [algorithms.md — Bresenham Frame Resampling](algorithms.md#bresenham-frame-resampling).

### Backend job hangs at 0%

ffmpeg or ffprobe is not on `$PATH`. The processor calls `subprocess.run(["ffmpeg", ...])` — if ffmpeg is missing, it raises `FileNotFoundError` which the background worker catches and marks the job as `failed`. Check the terminal where uvicorn is running for the traceback.

### TypeScript errors after adding a model

`TargetModel` in `dataset.ts` is a string literal union. If you added a new model ID but forgot to add it to the union, you will see `Type '"MYMODEL"' is not assignable to type 'TargetModel'`. Add the string to the union in `dataset.ts`.

### Caption batch only generates for the first file

This is the stale closure bug in `runCaptionBatch`. The results are being committed inside the loop with `handleUpdateCaption` instead of collected first and committed once with `handleUpdateCaptionBatch`.
